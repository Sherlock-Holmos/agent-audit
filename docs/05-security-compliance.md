# 安全与合规基线

## 1. 访问控制
- 业务接口默认需要 JWT（除登录/注册/健康检查）。
- 严禁业务服务公网直连，必须经 Gateway。

## 2. 凭据管理
- 禁止在代码库硬编码生产密钥。
- 通过环境变量或密钥管理系统注入敏感信息。
- 建议每 90 天轮换关键凭据（JWT Secret、DB 密码等）。

## 3. 传输与存储安全
- 外部访问强制 HTTPS（由 Nginx 终结 TLS）。
- 日志中不得输出明文密码、token、隐私数据。
- 个人信息字段按最小化原则采集与展示。

## 4. 接口安全策略
- 网关按用户与IP限流，防止暴力请求。
- 异步执行接口支持 `Idempotency-Key`，防止重复提交。
- 对外错误信息避免泄露内部实现细节。

## 5. 合规建议
- 审计日志保留策略：至少 180 天（按组织要求调整）。
- 变更需经过评审与留痕，禁止无审批生产变更。
- 定期执行依赖漏洞扫描与修复。
